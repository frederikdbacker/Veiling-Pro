## Wat is er gewijzigd in deze sessie (Fase 1.5)

### Migratie 0018 (commit `4017955`)

```sql
begin;
alter table auctions rename to collections;
alter table lots                   rename column auction_id to collection_id;
alter table auction_lot_types      rename column auction_id to collection_id;
alter table bid_step_rules         rename column auction_id to collection_id;
alter table auction_breaks         rename column auction_id to collection_id;
alter table auction_spotters       rename column auction_id to collection_id;
alter table client_auction_seating rename column auction_id to collection_id;
alter table auction_lot_types      rename to collection_lot_types;
alter table auction_breaks         rename to collection_breaks;
alter table auction_spotters       rename to collection_spotters;
alter table client_auction_seating rename to client_collection_seating;
commit;
```

Postgres updateert FK-constraints automatisch bij RENAME. Indices
behouden hun naam (cosmetisch verschil — bv. `idx_auction_spotters_auction`
verwijst nu naar `collection_spotters`). Functioneel werkend, voor
grep-baarheid eventueel later renamen.

### Code-changes (commit `4017955`)

**Page-files renamed:**
- `src/pages/AuctionPage.jsx` → `src/pages/CollectionPage.jsx`
- `src/pages/AuctionSummaryPage.jsx` → `src/pages/CollectionSummaryPage.jsx`
- Component-functienames bijgewerkt (`AuctionPage` → `CollectionPage`,
  idem voor SummaryPage)

**App.jsx routes:**
- `/auctions/:auctionId` → `/collections/:auctionId` (param-naam blijft
  `:auctionId` voor consistentie met JS-variabelen — bewust)
- `/auctions/:auctionId/summary` → `/collections/:auctionId/summary`
- `/cockpit/:auctionId` ongewijzigd
- Imports bijgewerkt naar nieuwe component-namen

**Sed-sweep over 14 files** met patronen:
- `from('auctions')` → `from('collections')`
- `from('auction_lot_types')` → `from('collection_lot_types')`
- `from('auction_breaks')` → `from('collection_breaks')`
- `from('auction_spotters')` → `from('collection_spotters')`
- `from('client_auction_seating')` → `from('client_collection_seating')`
- `auction_id` → `collection_id` (in alle contexts: 'auction_id' string,
  auction_id: object-key, .auction_id member-access)
- `auctions!collection_id` → `collections!collection_id` (PostgREST embed,
  na de eerste auction_id replacement)
- `/auctions/` → `/collections/` (URL-paden in JSX Link-components en
  in window.location.origin-strings)

**Files geraakt door sed:**
- src/pages/CollectionPage.jsx + CollectionSummaryPage.jsx
- src/pages/CockpitPage.jsx + LotPage.jsx + HousePage.jsx
- src/components/LotTypesSelector.jsx + LotTypeDropdown.jsx
       + BidStepRulesEditor.jsx + BidStepRulesPreview.jsx
- src/lib/spotters.js + breaks.js + clients.js
- scripts/import-lots.mjs + import-pedigree.mjs

**Handmatige fix na sed:**
- `LotPage.jsx`: `setAuction(data.auctions)` → `setAuction(data.collections)`.
  PostgREST geeft de embedded data terug onder de naam van de embedded
  table (na rename: `collections`). Was gemist door sed (geen
  `auction_id` in deze regel).

### Wat NIET veranderd (bewust, scope-controle)

- **`auction_houses` tabel + kolommen** — blijft staan, geen rename
- **JS-variabelnamen** `auctionId`, `auction`, `setAuction`, `auctionRes`,
  etc. — variabele namen mismatchen nu met tabelnaam (`collections`),
  cosmetische inconsistentie
- **URL-route param-naam** `:auctionId` — niet hernoemd naar `:collectionId`
  zodat bestaande `useParams()` destructures niet hoefden bij te werken.
  Resultaat: URL `/collections/{x}` met param-naam `auctionId` in code
- **UI-tekst "Veiling"** — overal user-facing nog "Veiling X" / "Veilingen"
- **Breadcrumbs** — geen nieuwe breadcrumbs toegevoegd (item #17 uit
  roadmap vraagt dat ook nog)
- **Indices in DB** — `idx_auction_spotters_*` etc. niet gerenamed

## Wat zou fout kunnen gaan

- **Mac mini desync** — als Frederik op de andere Mac werkt, moet hij
  `git pull` doen vóór hij iets queryt. Oude code daar zou nu falen
  (queryt `auctions`-tabel die niet meer bestaat).
- **Vercel-deploy lag** — er was een 30-60s venster waarin productie
  kapot was tussen migratie-uitvoer (Stap 2) en Vercel-deploy. Ervaring:
  uitvoeren in deze volgorde houdt downtime minimaal.
- **scripts/import-pedigree.mjs** — REST API-URL-string is bijgewerkt
  (`collection_id=eq.${...}`), maar dit script wordt zelden gebruikt;
  nog niet end-to-end getest sinds de rename.
- **Cognitieve dissonantie** — `auctionId` variabele houdt nu een
  collection-id; future-Frederik (of future-Claude) zou kunnen denken
  dat dit een aparte concept is. Audit-vermelding hiervan + PROJECT_STATUS-
  update zal helpen.
- **Indices/RLS-policies** — niet expliciet ge-audit. Postgres
  hernoemt FK's automatisch maar named indices niet. RLS-policies
  zouden bij table-rename automatisch mee hernoemd moeten zijn maar
  niet geverifieerd.
- **CockpitStatusBar.jsx en andere helper-components** — de sed-sweep
  raakte 14 files; eventuele andere components die `auctions`/`auction_id`
  refereerden zouden gemist zijn. Build-success suggereert dat alle
  references in build-tijd OK zijn, maar runtime PostgREST-error blijft
  mogelijk.
